Phase 2: AI Agent 验证（第一块试验田）
--------------------------------------
目标：用 SMS 制造 AI Agent 组件，证明标准体系可用。

Agent 的 5 类标准件：
  Tool    - 工具能力模块
  Memory  - 记忆/存储模块
  Planner - 规划/推理模块
  Executor - 执行模块
  Protocol - Agent 间通信协议模块

验证目标：
  制造一个双 Agent 协作系统
  Agent A（需求分析） + Agent B（代码生成）
  证明：SMS 标准在 Agent 领域能产生数量级效率提升

关键任务：
  [ ] 定义 Agent 领域的 Capability 标准词汇
  [ ] 实现 Agent 行为验证（Evidence for Agent）
  [ ] 实现多 Agent 装配拓扑（主从/对等/流水线）
  [ ] 端到端 Demo：图谱定义 -> 模块制造 -> 装配 -> 运行


Phase 3: 第二个领域验证
-----------------------
候选领域：
  - 编译器组件（Lexer、Parser、CodeGen）
  - 后端服务（Auth、Cache、Queue）
  - CLI 工具链

目标：证明标准可跨领域迁移


Phase 4: 标准化扩散
-------------------
  [ ] 开放 Concept Registry 扩展机制
  [ ] 多语言 Backend 完善（Rust、JavaScript、Java）
  [ ] 社区贡献 Importer/Exporter
  [ ] 模块市场 / Registry 服务


核心原则
--------
1. 不发明新东西，借鉴工业世界已验证的标准
2. 先在足够重要的领域证明价值，再扩展边界
3. AI 只是供应商之一，工厂才是核心
4. 标准模块是递归的——组合后的产品仍是模块
