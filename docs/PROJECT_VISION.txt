SMS (Software Manufacturing System) 项目愿景
================================================

一句话说清 SMS 是什么
--------------------
SMS 不是 IDE，不是编译器，不是编程语言，不是 AI 编程助手。
SMS 是一个软件制造系统——用工业化方式生产软件。


核心思想
--------
工业革命前，工匠手工制作每一个零件。
工业革命后，标准件 + 流水线 + 工厂 = 规模化制造。

SMS 做的是同样的事，对象是软件：
AI 时代之前，程序员一行一行写代码。
AI 时代之后，AI 生产模块，工厂装配产品。


软件的第一公民不是代码
---------------------
代码是化石，决策是回应，问题是根源。

没有"共享文件的问题"，就没有 Unix。
没有"网络互联的问题"，就没有 TCP/IP。
没有"版本管理的问题"，就没有 Git。

SMS 保存的不是代码，是软件之所以成为软件的认知根源：
Problem -> Decision -> Module -> Product


核心理念
--------
1. 软件模块 = 电子元器件
   借鉴电子元器件行业的规格书、品控、封装标准

2. 工厂不关心原材料来源
   手写代码、AI 生成、GitHub 导入、第三方库
   进入工厂后全部变成统一的标准模块

3. 模块是递归的
   一个函数可以是模块，一个类可以是模块，
   一个包可以是模块，一个完整产品也可以是模块。
   模块与大小无关，只与是否有清晰契约有关。


SMS 不是什么
-----------
不是库管理器（npm、pip、cargo 是管理包，SMS 是制造模块）
不是 AI 编程助手（AI 只是工厂的供应商之一，不是工厂本身）
不是新的编程语言（SMS 输出的是标准模块，Python/Rust/Java 只是后端适配）


长期愿景
--------
建立软件世界第一套真正可交换、可验证、可制造的标准件体系。
今天的软件行业没有真正的"零件标准"——Python 模块、Rust crate、
Java jar、C++ 库都是语言级标准，不是能力级标准。
SMS 的目标是建立跨语言、跨平台的能力级标准。
